CruiseControl.NET : Trigger Blocks
This page last changed on Jan 08, 2005 by mroberts.

Trigger blocks allow you to specify when CruiseControl.NET will start a new integration cycle.

Typically, you'll want to use an Interval Trigger. Also useful is the Schedule Trigger for implementing daily builds. Use the Filter Trigger to prevent builds from occurring at certain times (such as when your source control repository is undergoing backup).

Types of Integration Trigger Block are:

  • Filter Trigger
  • Interval Trigger
  • Schedule Trigger

NOTE: Force Build Interval Trigger and Polling Interval Trigger have been replaced by Interval Trigger, and the Force Build Schedule Trigger and Polling Schedule Trigger have been replaced by Schedule Trigger, from release 0.8 and onwards.
