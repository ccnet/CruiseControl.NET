User Name Authentication

User name authentication checks that just the user name is valid.

Note: This authentication performs no checks other than the user name is valid

Version

This security item is available from version 1.5.0.

Examples

Simple example: 

<simpleUser name="johndoe" display="John Doe"/>

Wildcard example:

<simpleUser name="*"/>

Configuration Elements

Node Description Type Required Default Version
name The name of the user. See note on wild cards below. String Yes n/a 1.5.0
display A display name for the user. If this is missing then the name will be used for the display name. String No name 1.5.0

Wildcards in User Names

It is possible to use wild cards for the user name. To use a wild card simply use an '*'. Wild cards can either be the whole name (in which case everything will be matched) or just part of a name.

Some examples are:

* matches everything
local/* matches every name from the local domain
*@local.com matches all the names with e-mail addresses from local.com
