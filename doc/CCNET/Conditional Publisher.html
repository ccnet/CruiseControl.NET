Conditional Publisher

Runs the child publishers only if the condition is met.

Currently the only available condition that can be checked is the state of the build.

Version

This task is available from version 1.5.0.

Examples

Minimalist example:

<conditionalPublisher>
  <conditions>
    <condition>Success</condition>
  </conditions>
  <publishers>
    <!-- Add publishers here -->
  </publishers>
</conditionalPublisher>

Full example:

<powershell>
  <conditions>
    <condition>Success</condition>
  </conditions>
  <publishers>
    <!-- Add publishers here -->
  </publishers>
  <description>Example of how to include a conditional publisher.</description>
</powershell>

Configuration Elements

Node Description Type Required Default Version
publishers The publishers to run if the conditions are met. Publisher array - see Task Blocks Yes n/a 1.5.0
conditions A list of conditions, or which one must be met in order to run the publishers. Status list:
  • Success
  • Failure
  • Exception
  • Unknown
  • Cancelled
 Yes n/a 1.5.0
description A description of this task. String No None 1.5.0
dynamicValues Any dynamic parameters to be used. See Dynamic Parameters Dynamic values array - see Dynamic Parameters No None 1.5.0
