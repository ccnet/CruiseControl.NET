Highlights of the CCNet 1.4.1 release

The highlights have been amended
The highlights for 1.4.1 previously said that the reporting graphs now worked in Firefox 3. That was incorrect. The work to update Dojo was not completed for the 1.4.1 release. Apologies for the confusion.
Category shown on Dashboard

The category element is now used in the Dashboard. Projects are now grouped per category, if you select the view per buildserver.

History graph at project level

There is a now also a graphical build history in the project overview.

Links to issue trackers in commit comments

Comments can be transformed into urls pointing to issue tracker webpages if they follow a pattern, this can be done by the using the issueUrlbuilder in a source control block.

Passwords masked on Dashboard

The ProjectView configuration now masks the passwords by default.

All JIRA issues reviewed

Daniel Hommel reviewed all the issues recorded in JIRA. Issues which are not reproducible in the 1.4 version have been marked as fixed. This will help the team more clearly see what remains to be done. This also means many of the issues listed as fixed in this release are the results of changes made in earlier releases.
