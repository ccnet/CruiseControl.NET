Date Parameter

This will prompt the user to enter a date value when a force build is requested.

This parameter can then be used by a dynamic value in a task.

Version

This parameter is available from version 1.5.0.

Examples

Minimal example:

<dateParameter>
  <name>CutOffDate</name>
</dateParameter>

Full example:

<dateParameter>
  <name>CutOffDate</name>
  <display>Cut Off Date</display>
  <description>What is the cut-off date for changes?</description>
  <default>today</default>
  <minimum>1-Jan-2000</minimum>
  <maximum>31-Dec-2100</maximum>
  <required>false</required>
</dateParameter>

Configuration Elements:

Node Description Type Required Default Version
name The name of the parameter. This is the name that will be used internally by CruiseControl.NET. String Yes n/a 1.5.0
display The display label for the parameter. If set, this is what the user will see. If it is not set, then the name will be used for the label. String No none 1.5.0
description A description to display to the user. String No none 1.5.0
default The default value to use if the user does not enter a value for this parameter. String No None 1.5.0
minimum The minimum allowed date. Date No 1-Jan-0000 1.5.0
maximum The maximum allowed date. Date No 31-Dec-9999 1.5.0
required Whether the parameter is required or not. Boolean No False 1.5.0

Default Values

The following special values are allowed for the default value:

Value Description Example
today Today's date today
dayofmonth The day of the month. This must be followed by a number in brackets for the day. dayofmonth(15)
dayofweek The day of the week, where 0 is Sunday through to 6 is Saturday. This must be followed by a number in brackets for the day. dayofweek(3)

Additionally, it is possible to add or subtract days from one of the special values. E.g. today-3 would be the date three days ago.
