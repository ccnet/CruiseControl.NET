Assembly Version Labeller

Uses the CruiseControl.NET change number, provided by source control systems like Subversion, to build a valid Assembly Version label.

New feature

This labeller is only available from release 1.4.4 onwards.

Examples:

Minimalist Example:

<project>
   <labeller type="assemblyVersionLabeller" />
   <!-- Other project configuration to go here -->
</project>

Full Example:
<project>
   <labeller type="assemblyVersionLabeller">
      <major>$(majorVersion)</major>
      <minor>$(minorVersion)</minor>
      <build>$(buildVersion)</build>
      <revision>$(revisionVersion)</revision>
      <incrementOnFailure>false</incrementOnFailure>
   </labeller>
   <!-- Other project configuration to go here -->
</project>

Configuration Elements:

Node Description Type Required Default
major Major number component of the version. int No 0
minor Minor number component of the version. int No 0
build Build number component of the version. int No -1
revision Revision number component of the version. int No -1
incrementOnFailure Whether to increase the label if the build fails. By default the label will only increase if the build is successful. boolean No false
