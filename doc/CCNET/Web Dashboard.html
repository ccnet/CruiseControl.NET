CruiseControl.NET : Web Dashboard
This page last changed on Aug 16, 2005 by mroberts.

The CCNet Web Dashboard Application is used for reporting a wide range of information. At one end of the scale it reports summary details of all projects in your organisation and at the other it can give specific metric output for any specific build.

Features of the Web Dashboard include:

  • Multi-project, multi-server support - 1 Web Dashboard deployment can report over all the CCNet projects in your organisation so there is no need for multiple instances of a web application.
  • A significant set of 'plugins' to support various reporting and configuration features
  • Plugin API supports custom features at various groupings of build, project, etc.
  • Plugin API supports a complete code interface, enabling significantly richer plugins than were available with the old Web Application.

  • If you want to start off administering the Web Dashboard, read Installing the Web Dashboard.
  • Read Configuring the Web Dashboard to specify Dashboard behaviour, and service locations.
  • Read Using the Web Dashboard if you want to actually work with the Dashboard.
  • Read Developing Web Dashboard Plugins if you're interested in extending the scope of features of the Dashboard.
  • Read the Web Dashboard FAQ for some common solutions.
